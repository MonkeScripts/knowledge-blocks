01

Architecture

Kubernetes (K8s) is a container orchestration platform that automates deployment, scaling, and management of containerized applications. Declarative configuration — you describe desired state, K8s makes it happen. Control plane manages cluster state, worker nodes run containers.

Control Plane

API server, etcd (state store), scheduler (assigns pods to nodes), controller manager (reconciliation loops). The brain of the cluster.

Worker Nodes

Run kubelet (node agent), kube-proxy (networking), and container runtime (containerd/Docker). Where your pods actually execute.

kubectl

CLI tool to interact with the cluster. kubectl apply -f submits manifests, kubectl get pods checks status. Essential daily driver.

Namespaces

Virtual clusters within a cluster. Isolate teams, environments (dev/staging/prod), or applications. Resource quotas per namespace.

02

Workloads

Pods are the smallest deployable unit — one or more containers sharing network and storage. You rarely create pods directly. Deployments manage pod replicas with rolling updates and rollbacks. StatefulSets for databases, DaemonSets for per-node agents, Jobs for batch work.

deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: my-app
spec:
  replicas: 3
  selector:
    matchLabels:
      app: my-app
  template:
    spec:
      containers:
      - name: app
        image: my-app:1.2.0
        ports:
        - containerPort: 8080
        resources:
          requests:
            memory: "128Mi"
            cpu: "250m"
Tip: Always set resource requests and limits. Without them, a runaway pod can starve the entire node.
03

Services & Networking

Services provide stable network endpoints for pods. ClusterIP for internal traffic, NodePort for external access, LoadBalancer for cloud LBs. Ingress handles HTTP routing — hostname and path-based rules pointing to services.

ClusterIP

Default service type. Internal-only virtual IP. Other pods reach it by service name via cluster DNS.

Ingress

HTTP/HTTPS routing layer. Map api.example.com to one service, app.example.com to another. TLS termination, path rewrites.

Use Cases

Auto-Scaling Microservices
scaling / resilience / load handling
scaling microservices

Horizontal Pod Autoscaler watches CPU/memory metrics and adjusts replica count. Cluster Autoscaler adds/removes nodes. Handle traffic spikes without manual intervention.

🔄
Zero-Downtime Deployments
rolling updates / canary / blue-green
deploy reliability

Rolling update strategy replaces pods incrementally. Readiness probes ensure new pods handle traffic before old ones terminate. Rollback with kubectl rollout undo if metrics degrade.
