01

Core Concepts

Continuous Integration — every commit triggers automated build + test. Catch bugs at merge time, not in production. Continuous Delivery — every passing build is deployable. Continuous Deployment — every passing build deploys automatically. CI is table stakes; CD is the competitive advantage.

Pipeline

Sequence of stages: checkout → install → lint → test → build → deploy. Each stage gates the next. Fail fast — lint before expensive tests.

Artifacts

Build outputs (Docker images, binaries, bundles) stored in registries. Tagged with commit SHA for traceability. The deployable unit.

Environments

Promotion chain: dev → staging → production. Each environment has its own config (secrets, URLs, feature flags). Staging mirrors prod.

Secrets

API keys, tokens, certificates stored in CI system's secret store. Injected as environment variables at runtime. Never in code, never in logs.

02

GitHub Actions

Workflow files in .github/workflows/. Triggered by events (push, PR, schedule, manual). Jobs run on GitHub-hosted or self-hosted runners. Steps execute commands or reusable actions from the marketplace.

.github/workflows/ci.yml
name: CI
on:
  push:
    branches: [main]
  pull_request:

jobs:
  build-and-test:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-node@v4
      with:
        node-version: 20
        cache: npm
    - run: npm ci
    - run: npm test
    - run: npm run build
Tip: Cache dependencies between runs. actions/setup-node with cache: npm skips npm ci when lockfile unchanged — saves 30-60s per run.
03

Deploy Strategies

How you roll out changes determines your risk tolerance. Rolling updates are the default. Canary and blue-green give more control at the cost of complexity.

Rolling Update

Replace instances gradually. Old and new versions coexist briefly. Simple, built into Kubernetes. Risk: breaking changes hit partial traffic.

Blue-Green

Run two identical environments. Deploy to inactive (green), test, swap traffic. Instant rollback = swap back. Cost: double infrastructure.

Canary

Route small percentage of traffic to new version. Monitor metrics. Gradually increase if healthy. Most controlled, most complex to implement.

Feature Flags

Deploy code dark, enable features per user/group. Decouple deployment from release. Rollback = flip flag, no redeploy needed.

Use Cases

Docker Image Pipeline
build / tag / push / deploy
docker registry

CI builds Docker image on every push, tags with sha-abc1234, pushes to container registry. CD updates Kubernetes deployment manifest with new image tag. GitOps: ArgoCD watches the manifest repo and syncs automatically.

🧪
PR Preview Environments
testing / review / collaboration
preview review

Each pull request spins up a temporary environment with the branch deployed. Reviewers test the actual running app, not just read diffs. Environment destroyed on PR merge/close.
