01

What Are Knowledge Blocks

A knowledge block is a deep reference document on one specific topic. It is NOT a skills chart, proficiency list, or shallow overview. A block is something you open, read the full writeup, check related sources, and use in future projects. Think of it as a personal wiki entry.

Good Block Topics

Specific, referenceable: Zenoh, CAN Bus Protocol, PX4 Offboard Control, Docker Multi-Stage Builds

Bad Block Topics

Too broad or vague: Embedded Protocols (category), Languages (not a topic), Skills Overview (not a reference)

Tip: If you can't write at least 3 detailed sections about a topic, it's probably too narrow for a block. If you need more than 10 sections, consider splitting into multiple blocks.
02

HTML Structure

Every guide resource file at public/resources/{slug}.html follows the same HTML class structure. The portfolio CSS handles all styling — never add inline styles or style tags to resource files.

Each block needs 4 files:

1. Resource HTML

public/resources/{slug}.html — the full guide content using the CSS classes documented here

2. Entry Page

blocks/{slug}.html — minimal HTML that loads block-detail.js with a data-block-slug attribute

3. Data Entry

src/shared/data.js — add slug, title, icon, description, tags, and sources to the knowledgeBlocks array

4. Vite Config

vite.config.js — add 'block-{slug}': resolve(root, 'blocks/{slug}.html') to rollup inputs

resource HTML skeleton
<!-- Hero -->
<div class="hero">
  <h1>Title<br>Subtitle</h1>
  <div class="subtitle">// tagline</div>
</div>

<div class="container">

  <!-- Optional nav pills -->
  <div class="nav">
    <a href="#section-id">section name</a>
  </div>

  <!-- Sections -->
  <div class="section" id="section-id">
    <div class="section-header">
      <span class="section-number">01</span>
      <h2>Section Title</h2>
    </div>
    <p class="prose">Content here</p>
  </div>

  <!-- Divider before use cases -->
  <div class="divider"></div>

  <!-- Use cases -->
  <div class="section" id="usecases">
    ...
  </div>

</div>
Example: data.js entry
src/shared/data.js
{

03

Using the Claude Code Skill

The fastest way to create blocks is with the /knowledge-block skill in Claude Code. Pass it a URL, file path, or raw text and it handles everything — reading the source, creating all 4 files, and verifying the build.

usage
# From a URL
/knowledge-block https://docs.docker.com/build/building/multi-stage/

# From a local file
/knowledge-block ~/notes/can-bus-protocol.html

# The skill will:
# 1. Read the source data
# 2. Check for existing blocks on the same topic
# 3. Create new block OR suggest merging into existing
# 4. Run npm run build to verify
Tip: The skill checks for duplicate topics automatically. If you pass Zenoh data and a Zenoh block already exists, it will suggest merging instead of creating a duplicate.

Use Cases

🎓
Learning Journal
education / self-study / reference
notes learning reference

Create blocks as you learn new topics. Each block becomes a permanent reference you can revisit and share. The "Share with Claude" button exports structured markdown you can paste into any AI conversation for context.

🛠️
Project Documentation
engineering / architecture / decisions
docs architecture decisions

Document the technologies and patterns you use across projects. When you need to set up the same tool again, the block has your exact configuration and lessons learned.

🤝
Team Knowledge Sharing
collaboration / onboarding / handoff
team onboarding

Deploy your knowledge blocks as a static site. New team members can browse your reference documents to understand the tools and patterns your team uses.
